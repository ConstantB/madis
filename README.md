 <h1><a name="About"></a>About<a href="#About" class="section_anchor"></a></h1><p><a href="http://doc.madis.googlecode.com/hg/index.html" rel="nofollow">madIS</a> is an extensible relational database system built on top of the <a href="http://www.sqlite.org/" rel="nofollow">SQLite</a> database with extensions implemented in Python (via <a href="http://code.google.com/p/apsw/" rel="nofollow">APSW</a> SQLite wrapper).  </p><p>In usage, madIS, feels like <strong>Hive</strong> (<strong>Hadoop</strong>&#x27;s SQL with User Defined Functions language), without the overhead but also without the distributed processing capabilities. Nevertheless madIS can easily handle tens of millions of rows on a single desktop/laptop computer. </p><p>madIS&#x27; main goal is to promote the handling of data related tasks within an extended relational model. In doing so, it upgrades the database, from having a support role (storing and retrieving data), to being a full data processing system on its own. madIS comes with functions for file import/export, keyword analysis, data mining tasks, fast indexing, pivoting, statistics and workflow execution. </p><p>madIS was designed and implemented by a small team of developers at the MaDgIK lab of the University of Athens, under the supervision of professor Yannis Ioannidis. </p><h1><a name="News"></a>News<a href="#News" class="section_anchor"></a></h1><ul><li>2013-05-14 - Version 1.7 is released: </li><ul><li>Many bug fixes and optimizations. madIS works better now with SQLite versions before 3.7.11. </li><li>Major cleanup/simplification of the Virtual Table operators. Most of the operators have been converted to using <strong>vtbase.py</strong>. </li><li>Added <a href="http://doc.madis.googlecode.com/hg/vtable.html#module-functions.vtable.unindexed" rel="nofollow">UNINDEXED</a> virtual table, which forces SQLite engine to access its embedded query without using indexes. </li><li>Added <a href="http://doc.madis.googlecode.com/hg/vtable.html#module-functions.vtable.ordered" rel="nofollow">ORDERED</a> virtual table, which hints to SQLite engine that the embedded query&#x27; results are already sorted, so it can do a <strong>GROUP BY</strong> operation on these, without constructing an index on them first. </li><li>Added functions userfull for text mining. These are <a href="http://doc.madis.googlecode.com/hg/row.html#module-functions.row.text" rel="nofollow">TEXTWINDOW</a>, which produces a rolling window over an input text, <a href="http://doc.madis.googlecode.com/hg/row.html#module-functions.row.text" rel="nofollow">REGEXPCOUNTWITHPOSITIONS</a> and <a href="http://doc.madis.googlecode.com/hg/row.html#module-functions.row.text" rel="nofollow">REGEXPCOUNTWORDS</a>, which match regular expression patterns to text in different ways. </li></ul></ul><ul><li>2012-06-08 - Version 1.6 is released: </li><ul><li>Bug fixes to madIS I/O engine. madIS I/O engine is now up to 3 times faster. </li><li>Added a line based JSON streaming format to <a href="http://doc.madis.googlecode.com/hg/vtable.html#module-functions.vtable.output" rel="nofollow">OUTPUT</a> and <a href="http://doc.madis.googlecode.com/hg/vtable.html#module-functions.vtable.file" rel="nofollow">FILE</a>. The JSON format, by default, preserves schema and type info of data. </li><li>Added &quot;strict&quot; option to <a href="http://doc.madis.googlecode.com/hg/vtable.html#module-functions.vtable.file" rel="nofollow">FILE</a> that works in the same manner as <a href="http://doc.madis.googlecode.com/hg/vtable.html#module-functions.vtable.xmlparse" rel="nofollow">XMLPARSE</a>&#x27;s one. </li><li>Added <a href="http://doc.madis.googlecode.com/hg/row.html#module-functions.row.date" rel="nofollow">DATE2ISO</a> row function. The function autodetects the format of an input date and converts it to ISO8601 date format. </li></ul></ul><ul><li>2012-03-16 - Version 1.5 is released: </li><ul><li>More robust <a href="http://doc.madis.googlecode.com/hg/vtable.html#module-functions.vtable.xmlparse" rel="nofollow">XMLPARSE</a>. </li><li>Added <a href="http://doc.madis.googlecode.com/hg/row.html#module-functions.row.util" rel="nofollow">EXECPROGRAM, GZ and UNGZ</a> utility functions. </li><li>Added <a href="http://doc.madis.googlecode.com/hg/aggregate.html#module-functions.aggregate.graph" rel="nofollow">GRAPHPOWERHASH and GRAPHTODOT</a> functions. GRAPHPOWERHASH is an experimental function that can be used for graph isomorphism (such as finding same chemical molecules) and graph similarity problems (such as finding similar molecules or same molecule parts). </li><li><a href="http://doc.madis.googlecode.com/hg/aggregate.html#module-functions.aggregate.statistics" rel="nofollow">PEARSON, STDEV and VARIANCE</a> functions now use arbitrary precision fraction arithmetic for maximum calculation accuracy. </li><li>Added harvesting and processing examples. The examples are based on data/services from <a href="http://code.google.com/p/madis/source/browse/#hg%2Fsrc%2Fexamples%2Fdblp" rel="nofollow">DBLP</a>, <a href="http://code.google.com/p/madis/source/browse/#hg%2Fsrc%2Fexamples%2Fciteseer" rel="nofollow">CiteSeerX</a> and <a href="http://code.google.com/p/madis/source/browse/#hg%2Fsrc%2Fexamples%2Fdiavgeia" rel="nofollow">DIAVGEIA</a>. </li><li><a href="http://doc.madis.googlecode.com/hg/vtable.html#module-functions.vtable.output" rel="nofollow">OUTPUT</a> can now directly export to one (or split to many) external SQLite DB. </li><li>Added <a href="http://doc.madis.googlecode.com/hg/row.html#module-functions.row.text" rel="nofollow">HASHMD5MOD and CRC32</a> functions. </li></ul></ul><ul><li>2011-12-08 - Version 1.4 is released: </li><ul><li><a href="http://doc.madis.googlecode.com/hg/vtable.html#module-functions.vtable.xmlparse" rel="nofollow">XMLPARSE</a> can now work without any provided prototype, producing JSON dicts which contain all parsed XML data. XMLPARSE accepts now Jdicts and Jlists in addition to XML snippets as prototypes. </li><li><a href="http://doc.madis.googlecode.com/hg/vtable.html#module-functions.vtable.file" rel="nofollow">FILE</a> works with gzip compressed files, HTTP and FTP streams directly in a streaming way. </li><li>New functions which work with Jdicts were added (jdictkeys, jdictvals, jdictsplit, jdictgroupunion). </li><li><a href="http://doc.madis.googlecode.com/hg/row.html?highlight=apachelog#functions.row.logparse.apachelogsplit" rel="nofollow">APACHELOGSPLIT</a> parses and splits Apache log lines. </li><li>Optimizations in Virtual Tables (up to 3 times faster). XMLPARSE is up to 2x faster (using <i>fast:1</i> switch). </li></ul></ul><ul><li>2011-07-18 - Version 1.3 is released: </li><ul><li><a href="http://doc.madis.googlecode.com/hg/vtable.html#module-functions.vtable.xmlparse" rel="nofollow">XMLPARSE</a> was added. XMLPARSE processes its input in a streaming fashion, and has been tested with very large (~20+ GB) XML source files without problems. </li><li><a href="http://doc.madis.googlecode.com/hg/row.html#module-functions.row.jpacks" rel="nofollow">JPACK functions</a> were added. Jpacks are now the preferable way to store a set of values into a single tuple. For easy viewing and exporting of the jpacks, their format was based on the <a href="http://www.json.org" rel="nofollow">JSON</a> format. </li><li>Heavy testing under Windows and Mac OSX. <a href="http://doc.madis.googlecode.com/hg/vtable.html#module-functions.vtable.clipboard" rel="nofollow">CLIPBOARD</a> and <a href="http://doc.madis.googlecode.com/hg/vtable.html#module-functions.vtable.clipout" rel="nofollow">CLIPOUT</a> virtual tables work under all OSes. </li><li>CLIPOUT and CLIPBOARD, have been tested with Excel, Libre/Open Office Calc, and iWork Numbers. </li><li>Functions that return tables, can easily be coded now, by using Python&#x27;s generators (yield) </li><li>A lot of completions (via TAB) have been added to mterm. Mterm&#x27;s completion engine can automatically complete, tables, column names, table index names and database filenames in <strong>attach database</strong>. </li></ul></ul><ul><li>2010-07-09 - Version 1.2 is released: </li><ul><li><a href="http://doc.madis.googlecode.com/hg/vtable.html#module-functions.vtable.webtable" rel="nofollow">WEBTABLE</a> and <a href="http://doc.madis.googlecode.com/hg/vtable.html#module-functions.vtable.clipboard" rel="nofollow">CLIPBOARD</a>(untested under windows) virtual tables were added.  </li><li>The PEARSON correlation function was added in the <a href="http://doc.madis.googlecode.com/hg/aggregate.html#module-functions.aggregate.statistics" rel="nofollow">aggregate statistics functions</a> </li><li>Various other fixes. </li></ul></ul><ul><li>2010-04-29 - Version 1.1 is released: </li><ul><li><a href="http://doc.madis.googlecode.com/hg/vtable.html#module-functions.vtable.file" rel="nofollow">FILE</a> function can read now, non-UTF8 files via &#x27;encoding&#x27; parameter. </li><li><a href="http://doc.madis.googlecode.com/hg/vtable.html#module-functions.vtable.cache" rel="nofollow">CACHE</a> function, requires less memory now. </li><li>Various fixes. </li></ul></ul><h1><a name="madIS_is_suitable_for"></a>madIS is suitable for<a href="#madIS_is_suitable_for" class="section_anchor"></a></h1><h2><a name="Complex_data_analysis_tasks"></a>Complex data analysis tasks<a href="#Complex_data_analysis_tasks" class="section_anchor"></a></h2><p>With madIS it is very easy to create additional relational functions, or join against external files without first importing them into the database. </p><p>madIS also offers a very fast multidimensional index that greatly speeds up multi-constraint joins, even when joining against external sources. </p><h2><a name="Data_transformations"></a>Data transformations<a href="#Data_transformations" class="section_anchor"></a></h2><p>madIS can already use the file system or network sources as tables. In addition, with a little knowledge of Python, complex data transformation functions can be created and used inside the database. All these can be achieved without having to import the data in the database. </p><p>In addition madIS offers an easy to work with, workflow engine that automates data transformation steps. </p><h2><a name="Database_research"></a>Database research<a href="#Database_research" class="section_anchor"></a></h2><p>You can easily develop and test a new indexing structure with madIS, as it uses Python for its extensions and already has plenty of code to start from. </p><h1><a name="Documentation"></a>Documentation<a href="#Documentation" class="section_anchor"></a></h1><p>You&#x27;ll find madIS <a href="http://doc.madis.googlecode.com/hg/index.html" rel="nofollow">documentation here</a>. </p><h1><a name="Download"></a>Download<a href="#Download" class="section_anchor"></a></h1><p>You can download the latest madIS version from <a href="http://code.google.com/p/madis/downloads/list" rel="nofollow">madIS download page</a>. You should also take a look at <a href="http://doc.madis.googlecode.com/hg/install.html" rel="nofollow">madIS installation instructions</a>. </p><h1><a name="Screenshot"></a>Screenshot<a href="#Screenshot" class="section_anchor"></a></h1><p><img src="http://doc.madis.googlecode.com/hg/_static/madis-screen.png" /> </p>
